## Troubleshooting Philosophy

A structured troubleshooting process often resolves issues faster than immediately searching for technical solutions.

Typical workflow:

1. Define the problem clearly
2. Verify scope and impact
3. Collect evidence
4. Isolate possible causes
5. Test assumptions
6. Implement corrective action
7. Document findings
8. Prevent recurrence
